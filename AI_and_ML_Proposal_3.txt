




Market Price Forecasting for Ugandan Staple Crops A Multi-Mode Time-Series Approach Using WFP Market Data and Climate Covariates
Course Unit: Artificial Intelligence and Machine Learning
Programme: Master of Science in Computer Science (Year 1, Semester 2)
Institution: Uganda Christian University (UCU)

Date: April 2025

1. Problem Statement

Price volatility for staple crops - maize, beans, and cassava - causes food insecurity and income shocks for smallholder farmers and traders across Uganda. Despite the importance of these markets, no reliable data-driven forecasting tool currently provides district-level price projections 4-8 weeks in advance. Existing monitoring systems by WFP and FEWS NET are largely retrospective, reporting current prices rather than predicting future ones. This project addresses that gap by building and comparing three machine learning forecasting models - ARIMA, Facebook Prophet, and LSTM - trained on historical WFP market price data for Uganda, augmented with CHIRPS rainfall data as an external
covariate. The central research question is:

"How accurately can machine learning models forecast staple crop market prices 4 to 8 weeks ahead in Uganda, and which model - ARIMA, Prophet, or LSTM
- achieves the best performance when rainfall is incorporated as a predictor?"

2. Relevance

Agriculture contributes approximately 24% of Uganda's GDP and employs over 70% of the workforce. Early price signals allow farmers to decide when to sell or store produce, enable NGOs to intervene before food crises escalate, and support evidence-based policy planning by the Ministry of Agriculture. Academically, this project provides a reproducible benchmark comparison of classical and deep learning forecasting methods on

a real Ugandan dataset - a study not yet published at district-level granularity for East African staple crops.

3. Objectives

1. Collect, clean, and merge WFP historical price data with CHIRPS rainfall and fuel price indices for selected Ugandan districts.
2. Engineer lag features, seasonal indicators, and climate covariates as model inputs.

3. Implement and train three forecasting models: ARIMA, Facebook Prophet, and
LSTM.

4. Evaluate and compare model performance using MAE, RMSE, and MAPE on a held-out test set of recent months.
5. Deploy the best-performing model in an interactive web dashboard built with
Streamlit.

4. Methodology

Data Collection: Three datasets will be merged into a single analysis-ready dataframe: WFP Uganda Food Prices (HDX/Kaggle) providing weekly/monthly crop prices in UGX across districts; CHIRPS Rainfall Data (UC Santa Barbara) providing monthly rainfall in mm per district; and the World Bank Commodity Prices index as a fuel/transport cost proxy. Missing values will be handled via linear interpolation and date frequencies aligned to monthly.
   Feature Engineering: Lag features (price at t-1, t-2, t-4, t-8), rolling 4-week and 8-week statistics, month-of-year and harvest season dummies, and rainfall lags (t-4, t-8) will be engineered before training.
Models:

• ARIMA - classical statistical baseline; p, d, q selected via AIC/BIC grid search using statsmodels.
• Facebook Prophet - additive decomposition model with external regressors (rainfall); implemented via the prophet library.
• LSTM - two-layer recurrent neural network (64 units, dropout = 0.2, Adam optimizer) trained with TensorFlow/Keras on sliding-window sequences.
   All models will use an 80/20 chronological train/test split. Multi-step forecasting (4 and 8 weeks ahead) will use a recursive strategy for ARIMA/Prophet and a sequence-to- sequence approach for LSTM.

   Evaluation: Models will be compared using MAE (interpretable in UGX), RMSE (penalises large errors), and MAPE (scale-independent, enables cross-crop comparison), evaluated across 3 crops × 3 districts × 2 forecast horizons.
   Dashboard: A Streamlit web application will allow users to select a crop, district, and forecast horizon, and view predicted price trends alongside historical data, with a model comparison panel.
   Development Approach: Agile methodology - data sprint, modelling sprint, dashboard sprint over 3 weeks. Version control via GitHub with task allocation by team member.

5. Expected Outcomes

A functional forecasting system that:

• Produces 4-8 week ahead price forecasts for maize, beans, and cassava across Kampala, Gulu, and Mbale districts.
• Quantitatively compares ARIMA, Prophet, and LSTM using MAE, RMSE, and MAPE.
• Visualises forecasts and model performance in an interactive Streamlit dashboard.
• Generates a written report with methodology, results, and policy recommendations.
Deliverables include cleaned datasets, trained model artefacts, source code on GitHub, the dashboard, and a final project report with a live demo.

7. Scope and Feasibility

The project will be completed in three weeks. Crops covered: maize, beans, cassava. Districts: Kampala, Gulu, Mbale. Time period: 2010-2024. The minimum viable product (MVP) is a trained ARIMA baseline plus a Prophet model with MAE/RMSE comparison. LSTM and the full Streamlit dashboard are the stretch targets. The project demonstrates proficiency in time-series modelling, Python ML pipelines, and full-stack data application development. All datasets are freely available with no access restrictions.

8. Expected Impact

This project will deliver a reusable, open-source forecasting tool relevant to food security stakeholders in Uganda, including NGOs, the Ministry of Agriculture, and smallholder farming cooperatives. It promotes data-driven decision-making, advances applied ML research in the East African context, and contributes to Uganda Christian University's goal of producing graduates who solve real local problems with technology.
